The topic of today's post is on sparse affine transformations in NNUEs. It's a crucial technique for accelerating inference of multi-layer networks, and it goes surprisingly deep. It's possible to implement sparse affine in single layer networks, although it's much more effective in multi-layer networks due to the higher cost of the first layer after the feature transformer, and the lower number of nonzeros due to the pairwise product.

The idea of sparse affine is simple: we identify blocks of four zeros in the feature transformer output and skip them. We do it in blocks of fours as it works really well with how we perform the layer 1 matmul using dpbusd. These blocks of zeros don't contribute to the result of the matmul, thus sparse affine does not affect the output of our model, only the inference speed.

Writing the Sparse Affine Inference:

There are three steps in this process:
1. finding the nonzero blocks
2. getting the index into these nonzero blocks
3. doing the matmul

For the first step, there are some handy SIMD intrinsics we can use to vectorize this process: 

// AVX512: finds 16 nonzero blocks at a time
inline uint32_t nonzero_mask(__m512i reg) {
    return _mm512_cmpgt_epi32_mask(reg, _mm512_setzero_si512());
}

// AVX2: finds 8 nonzero blocks at a time
inline uint32_t nonzero_mask(__m256i reg) {
    return _mm256_movemask_ps(
        _mm256_castsi256_ps(_mm256_cmpgt_epi32(reg, _mm256_setzero_si256()))
    );
}
This blog shows how to implement nonzero_mask in other target arches such as SSSE3 and NEON, and also shows the full unrolled inference code.

For the second step, we convert these masks into indices. On AVX2, a single __m256i register contains 32 u8 values, corresponding to 8 blocks. The result of nonzero_mask is a 32-bit integer whose lower 8 bits store the occupancy mask for those 8 blocks. For example, if the mask is 0b1100_1001, then blocks 0, 3, 6, and 7 are nonzero.

Notice that each index can be recovered by repeatedly finding the position of the least-significant set bit. In other words, the next index is given by std::countr_zero(mask), after which we clear that bit with mask &= mask - 1. Repeating this process enumerates all nonzero indices in ascending order. The total number of nonzero blocks is simply the popcount of the mask.

To avoid recomputing these indices at runtime, we can precompute the index list for every possible mask byte and use the popcount to determine how many entries to read from the lookup table. We also maintain a running base offset equal to the number of nonzero blocks encountered so far. For example, if two consecutive __m256i registers both produce the mask 0b1100_1001, then the first register contributes indices 0, 3, 6, and 7, while the second contributes 8, 11, 14, and 15.

We can further optimize this by using SSE2 intrinsics to compute and store all the indices at once. The following code shows how we can precompute the nonzero indices and use that to track the nonzero blocks from a stream of inputs. Note that this code comes from Raphael but is originally based on the sparse affine implementation of Stormphrax (which itself might have taken inspiration from somewhere else). 

class SparseIterator {
private:
    uint16_t indices_[L1_SIZE / 4] = {};
    int32_t count_ = 0;

    __m128i offset_ = _mm_setzero_si128();

    // precompute nonzero_idx[mask][nnz_idx] = position of nonzero block
    alignas(16) static constexpr MultiArray<uint16_t, 256, 8> nonzero_idx = [] {
        MultiArray<u16, 256, 8> idx{};

        for (int32_t i = 0; i < 256; i++) {
            int32_t nnz = 0;
            for (u8 mask = i; mask != 0; mask &= mask - 1)
                idx[i][nnz++] = std::countr_zero(mask);
        }

        return idx;
    }();

public:
    // total number of nonzero blocks (nnz)
    int32_t count() const { return count_; }

    // index into ft output for a given nonzero
    int32_t index(int32_t nonzero_index) const {
        return indices_[nonzero_index];
    }

    // track nonzero blocks right after we activate them
    void add_nonzero(VecU8 ft_out0, VecU8 ft_out1) {
        // VecU8 = __m256i on AVX2, __m512i on AVX512
        // ALIGNMENT = 32 on AVX2 or 64 on AVX512
        constexpr int32_t regw32 = ALIGNMENT / sizeof(int32_t);
        constexpr int32_t n_mask_bytes = 2 * regw32 / 8;

        uint32_t full_mask = (nonzero_mask(ft_out1) << regw32) | nonzero_mask(ft_out0);
        for (int32_t i = 0; i < n_mask_bytes; i++) {
            // get offset of up to 8 nonzero blocks
            const uint8_t mask = full_mask & 0xFF;
            full_mask >>= 8;

            const auto idxs = _mm_add_epi16(
                offset_,
                _mm_load_si128(reinterpret_cast<const __m128i*>(&nonzero_idx[mask]))
            );

            _mm_storeu_si128(reinterpret_cast<__m128i*>(&indices_[count_]), idxs);
            offset_ = _mm_add_epi16(offset_, _mm_set1_epi16(8));
            count_ += std::popcount(mask);
        }
    }
};
We can further speed this up on CPUs with vbmi2 using vpcompress, the code for which you can find here. The code above takes in two registers as inputs as it works nicely with how vpcompress works, and also prevents accidental overflow of indices_ if our VecU8 is of type __m128i.

Lastly, we can use our SparseIterator class to speed up our layer 1 matmul. The following shows a rough idea of how this can be done. Note that in reality, we optimize this code by unrolling our accumulator registers by a factor of 4. You can see how this is done in Cosmo's blog or in my codebase. Anyways, here's the simplified inference code: 

SparseIterator sp;

{
    // ... pairwise computation above

    const VecU8 out0 = packus(pw0, pw1);
    const VecU8 out1 = packus(pw2, pw3);
    sp.add_nonzero(out0, out1);

    // ... store out0 and out1 into l0_out[L1_SIZE / 2]
}

constexpr regw32 = ALIGNMENT / sizeof(int32_t);
constexpr regw8 = ALIGNMENT / sizeof(int8_t);
constexpr n_chunks = L2_SIZE / regw32;
VecI32 l1_pre[n_chunks];

const int32_t nnz = sp.count();

for (int32_t i = 0; i < nnz; i++) {
    const int32_t tile_id = sp.index(i);
    const VecU8 inputs = tile_u8(&l0_out[4 * tile_id]);

    for (int32_t j = 0; j < n_chunks; j++) {
        const VecI8 weights = load_i8(&W1[bucket_idx][tile_id][j * regw8]);
        l1_pre[j] = dpbusd(l1_pre[j], inputs, weights);
    }
}

// ... activate l1 and do rest of inference

Permuting for Sparse Affine:

The above code already got me a 4% speedup compared to the dense affine on AVX2 but we can get this up to 8% by permuting our weights so that nonzero blocks are more likely to be grouped together.

The easiest way to do this is to track the number of activations in each of our ft output neurons and sorting in descending order of activations. I added an optional compiler flag in my Makefile to add a little bit of code to do this tracking, and permuted my ft weights, bias, and layer 1 weights in Python. 

def permute_sparsity(net: dict[str, np.ndarray]) -> dict[str, np.ndarray]:
    # we want to permute the l0 outputs and l1 inputs to group nonzeros
    with open("ft_activations.json", "r") as act_file:
        act: list[int] = json.load(act_file)
        order = np.argsort(act)[::-1]  # [L1_SIZE // 2]
        perm = np.concatenate([order, order + (L1_SIZE // 2)])  # [L1_SIZE]

    l0w = net["l0w"]  # [NUM_INPUT_BUCKETS, 12, 64, L1_SIZE]
    l0b = net["l0b"]  # [L1_SIZE]
    l1w = net["l1w"]  # [NUM_OUTPUT_BUCKETS][L2_SIZE][L1_SIZE]

    net["l0w"] = l0w[..., perm]
    net["l0b"] = l0b[perm]
    net["l1w"] = l1w[..., perm]
    return net
Doing this reduced my number of nonzero blocks from 59.76% to 46.09% with no changes in my output.

Since then, I've tried more sophisticated methods such as tracking co-activations to greedily form groups and doing 2-opt to further reduce my nnz, but it did not result in any measurable Elo gains. There are many ways to approach this problem, and perhaps you can find something that works better than these methods!

Training with Regularization:

Lastly, we can train our networks with a small regularization loss on our activations to further decrease our nnz. I just yoinked what other people use for their regularization loss and got up to a 12% speed improvement over the dense affine and a 13.04 Elo gain at VSTC. The average nnz blocks was 27.73%, and my failed 2-opt permutation improvements got it down to 25.78%.
